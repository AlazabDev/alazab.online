# إعداد خدمة Resend للبريد الإلكتروني

## المتطلبات
- حساب Resend (https://resend.com)
- API Key من Resend

## خطوات الإعداد

### 1. إنشاء حساب على Resend
1. اذهب إلى [https://resend.com](https://resend.com)
2. سجل حسابك الجديد
3. تحقق من بريدك الإلكتروني

### 2. الحصول على API Key
1. من لوحة التحكم (Dashboard)، اذهب إلى **API Keys**
2. انسخ مفتاح API الخاص بك
3. احفظه في مكان آمن

### 3. إضافة المفتاح إلى المشروع

#### في بيئة التطوير المحلية:
```bash
# انسخ ملف الأمثلة
cp .env.local.example .env.local

# ثم عدّل .env.local وأضف مفتاح API
RESEND_API_KEY=re_your_actual_api_key
```

#### في Vercel (الإنتاج):
1. اذهب إلى مشروع Vercel الخاص بك
2. اذهب إلى **Settings** → **Environment Variables**
3. أضف متغير جديد:
   - Name: `RESEND_API_KEY`
   - Value: `re_your_actual_api_key` (المفتاح الفعلي)
4. اضغط **Save**
5. أعد نشر المشروع

### 4. اختبار النظام
1. اذهب إلى صفحة التواصل في الموقع
2. أرسل رسالة اختبار
3. يجب أن تصل الرسالة إلى `info@al-azab.co`

## معالجة الأخطاء

إذا لم يكن API Key موجوداً:
- سيظهر للزائر رسالة: "البريد الإلكتروني غير متاح حالياً"
- يُنصح الزائر بالتواصل عبر الهاتف أو WhatsApp
- الموقع لن يتوقف عن العمل

## رسائل البريد المدعومة

### 1. رسالة التواصل (Contact)
- الاسم، البريد، الهاتف، الخدمة، الرسالة

### 2. طلب الاستشارة (Consultation)  
- الاسم، البريد، الهاتف، تفاصيل المشروع

## الأمان
- لا تضع API Key في الـ Git
- استخدم .env.local (مضاف إلى .gitignore)
- في الإنتاج، استخدم متغيرات Vercel البيئية
